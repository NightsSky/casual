# casual Product Brief

## 产品定位

casual 是一个基于 GitHub/Gitee 的本地优先笔记应用。用户可以在本地创建 TXT 或 Markdown 笔记，通过 Git 仓库同步到远程，并使用提醒助手管理周期性提醒。

## 目标用户

- 个人开发者、技术写作者、重度 Markdown 用户。
- 习惯用 Git 管理内容的人。
- 需要在 Windows 桌面和移动端之间同步笔记的人。

## 核心价值

- 本地优先：离线可用，编辑不依赖网络。
- Git 同步：通过 GitHub/Gitee 保存笔记文件。
- Markdown 友好：支持编辑、预览、格式转换和导出入口。
- 跨端适配：移动端底部导航，桌面端侧边栏和分栏编辑。
- 轻量提醒：支持单次、每天、每周、每月、工作日和间隔提醒。

## 重设计目标

1. 保留当前功能结构，不发明无关业务功能。
2. 提升开发者工具感，减少“普通表单 App”气质。
3. 优化笔记列表、编辑器、同步状态和设置表单的信息层级。
4. 移动端优先保证单手操作，桌面端强化分栏效率。
5. 降低暖棕色占比，加入更中性的背景与状态色，让界面更清爽。
6. 建立可复用组件：导航、列表项、状态标签、工具条、同步卡片、提醒弹窗、设置表单。
7. 保留专业克制，不做营销页、装饰性大卡片或过度插画。

## 不能丢失的业务状态

- 笔记同步状态：本地、已同步、冲突。
- 全局同步状态：未配置、同步中、成功、失败、上次同步时间。
- 编辑器模式：编辑、预览、Markdown 工具条、TXT/Markdown 格式转换。
- Windows 桌面独立窗口状态：已在新窗口打开、主编辑器只读、聚焦窗口。
- 提醒状态：启用/停用、单次/周期/间隔、删除确认。
- 设置状态：GitHub/Gitee、Token 隐藏、测试连接、保存配置、清空数据确认。
